# DL/T698.45 协议数据编辑传输可视化工具

> 基于 Python + tkinter 实现的 DL/T698.45 智能电表采集协议图形化工具，提供报文构造、收发、字段级解析与流量可视化能力，方便协议调试、学习与数据传输测试。

---

## 📌 项目简介

DL/T698.45 是电力行业新一代智能电表采集通信标准规约，广泛应用于台区集中抄表、用电数据采集、远程参数下发、电表密钥运维等现场业务，是低压电力物联网终端的核心交互协议。

本工具参照 **DL/T698.45-2017** 标准帧格式实现，以图形化界面完成协议报文的可视化构造与交互：

- 自定义目标 **IP / 端口**
- 多功能协议数据传输（**CONNECT / GET / SET / ACTION / RELEASE**）
- 报文自动编解码（**CRC16-FCS / HCS / 压缩 BCD**）
- 接收帧字段级解析与可视化展示
- 认证密码枚举测试（多线程，支持范围 / 字典）
- 全程流量日志

## 🧰 功能特性

### 1. 目标连接配置
顶部可配置目标 IP 与端口，支持建立 / 断开 TCP 长连接，连接状态实时显示。

### 2. 协议交互（多功能报文构造）
通过单选切换不同服务类型，自动构造合法 DL/T698.45 帧：

| 功能 | APDU 命令字 | 说明 |
|------|------------|------|
| CONNECT | `0x02` | 建立应用连接（认证类型 0~3 可选，6 位密码压缩 BCD 编码） |
| GET | `0x03` | 读取对象（自定义 4 字节 OID + PIID） |
| SET | `0x04` | 设置对象（OID + 属性 ID + 数据） |
| ACTION | `0x05` | 操作（OID + 方法 ID + 参数） |
| RELEASE | `0x01` | 断开应用连接 |

- 每帧自动计算并填充 **FCS**（整帧校验）与 **HCS**（帧头校验）
- 支持“仅构造预览”与“构造并发送”两种模式
- 接收响应自动解析：控制域位拆解（DIR/PRM/分帧/扰码/功能码）、APDU 命令、认证结果、OID、数据长度与明文等
- 必填项为空时弹出提示，无任何静默默认值

### 3. 原始帧发送
支持手动输入十六进制帧，可勾选“发送前自动重算 FCS/HCS”，便于协议调试与边缘报文测试。

### 4. 认证密码枚举测试
针对采用 6 位数字密码认证的场景，提供枚举测试能力：

- 支持 **6 位数字范围**（000000~999999）枚举或 **字典文件** 加载
- 多线程并发（线程数可调），可配单连接超时
- 每个候选密码新建 TCP 连接 → `CONNECT` 认证 → 判定结果 → 命中后**同一连接**发起 `GET` 读取目标对象数据
- 实时进度条 / 百分比 / 已尝试数显示
- 命中即停，自动展示密码与读取到的数据

### 5. 流量日志
独立 Tab 实时记录全部收发报文与解析提示，支持自动滚动。

## 📐 协议帧结构

依据 DL/T698.45-2017 第 5.1 节，工具实现的帧格式如下：

```
┌──────┬────────┬──────┬────────┬────────┬───────────────┬────────┬──────┐
│ 起始 │ 长度域 │ 控制 │ 地址域 │ 帧头   │  APDU / 分帧  │ 帧校验 │ 结束 │
│ 68H  │  L(2)  │  C   │  A(7)  │ HCS(2) │   用户数据    │ FCS(2) │ 16H  │
└──────┴────────┴──────┴────────┴────────┴───────────────┴────────┴──────┘
```

- **长度域 L**：2 字节，值为整帧除起始/结束符之外的长度
- **控制域 C**：客户机请求 `0x43`（DIR=0, PRM=1, 功能码=3 用户数据），服务器响应 `0xC3`
- **HCS**：覆盖 长度域 + 控制域 + 地址域
- **FCS**：覆盖整帧除结束符与 FCS 本身之外的所有字节
- **校验算法**：CRC16，多项式 `0xA001`，初值 `0xFFFF`，小端序

### CONNECT.Request APDU（23 字节）

```
apdu[0]    = 0x02  CONNECT
apdu[1..18]= 请求信息区
apdu[19]   = 认证类型 (1=普通密码)
apdu[20..22]= 3 字节压缩 BCD 密码
```

### GET.Request APDU（6 字节）

```
apdu[0]   = 0x03  GET
apdu[1]   = PIID
apdu[2..5]= 4 字节 OID
```

## 🚀 快速开始

### 环境要求

- Python 3.8+
- tkinter（Python 标准库，Linux 通常需安装 `python3-tk`）

> 无任何第三方依赖。

### 安装

```bash
git clone https://github.com/C-LBY/DL-T698.45-.git
cd DL-T698.45-
```

Linux 若缺少 tkinter：

```bash
sudo apt install python3-tk
```

### 运行

```bash
python3 dlt698_tool.py
```

### 典型用法

1. 顶部填写目标 IP 与端口
2. **协议交互** Tab 选择功能 → 填写参数 → 点击“构造并发送”，右侧查看响应与字段解析
3. **认证密码枚举** Tab 设置范围与线程数 → 开始测试，命中后自动展示密码与读取数据

## 🔬 技术要点

- **CRC16-FCS / HCS**：标准 `0xA001` 多项式，发送前自动重算，接收时自动校验
- **压缩 BCD 编码**：6 位数字密码 ↔ 3 字节 BCD 双向转换
- **同连接状态保持**：严格遵循“先认证、后读取，同一 TCP 连接内完成”的协议语义
- **多线程并发**：分片调度 + 命中即停，兼顾速度与资源占用
- **线程安全 UI**：收发与后台任务均在子线程执行，通过队列 + `after` 回主线程刷新界面，避免卡顿
- **无静默默认值**：必填项为空时明确提示，避免误操作
